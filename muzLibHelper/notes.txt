
База данных

Таблицы

Tracks:
    * ID
    * Name - имя, без всяких приписок, по типу ремкс и т.п. Приписки добавляются в поле Version
    * Version - расширенное имя, при писка по типу ремкс и т.п.
    * GroupID - связь с таблицей Groups
    * -GenreID - связь с таблицей Genres (СЕЙЧАС НЕ РЕАЛИЗОВАНО)
    * -TempID - связь с таблицей Temps (СЕЙЧАС НЕ РЕАЛИЗОВАНО)
    * -LanguageID - связь с таблицей Languages (СЕЙЧАС НЕ РЕАЛИЗОВАНО)
    * -Date - дата выпуска трека (СЕЙЧАС НЕ РЕАЛИЗОВАНО)
    
Примечания к таблице Tracks:
* суммарное имя (Name + Version) м.б. одинаковым у нескольких треков, их идентичность расрывается только вместе с полями авторов
* если у треков одинаковое суммарное имя (Name + Version) и одинаковые авторы, то они считаются дублями
    
Autors:
    * ID
    * Name - уникальное поле

TracksAutros:
    * ID
    * TrackID - связь с таблицей Tracks
    * AutorID - связь с таблицей Autors
    
TracksFiles:
    * ID
    * TrackID - связь с таблицей Tracks
    * Name - имя оригинального файла
    * InstrName - имя инструментального файла, его таймлайн д.б. совместим с оригинальным файлом
    * -Timeline - (СЕЙЧАС НЕ РЕАЛИЗОВАНО)

Примечания к таблице TracksFiles:
* Таблица может показаться, что таблица излишняя, но она сделана для удобства разделения базы треков и файлов.
* У одного трека м.б. болько один файл.

Groups:
    * ID
    * Name - уникальное поле
    
TracksRelationships:
    * ID
    * ParentTrackID - связь с таблицей Tracks
    * ChildrenTrackID - связь с таблицей Tracks
    * TypeID - связь с таблицей RelationshipsTypes
    
RelationshipsTypes:
    * ID
    * Type

Примечания к таблице RelationshipsTypes:
* поддерживаемые типы:
    * Remix - ремикс родительского трека
    * Format - другой формат трека (короче, длиньше, другие начало и конец, другое звучание(с кассеты, 8бит, ...))
    * NewVision - перепевка трека другим автором (не ремикс, т.к. меняется автор и стиль..)
    
    
Работа с БД:
* На основе таблицы TracksRelationships можно построить сущьность "супертрека", которая включает все однотипные перепевки, это пригодится при построении плейлиста, чтобы избежать повтора однотипных треков.

muzLibHelper

Особенности:
    * фиксированное позиционное расположение свойств
    * пропустить свойство - две точки (..)
    * разделитель свойств - двойной пробел (  )
    * разделитель множественного свойства - запятая (,)


Строка поиска:
(NAME [(VERSION)] | ..)  (AUTOR1[, AUTOR2, AUTOR3] | ..)  [FILE | ..]  [GROUP | ..]

    Найти все треки с таким именем:
    NAME  ..
    
    Найти все треки с такими авторами:
    ..  AUTOR1[, AUTOR2, AUTOR3]
    
    Найти все треки с таким файлом:
    ..  ..  FILE
    
    Найти все треки с такой группой:
    ..  ..  ..  GROUP
    
    Найти все треки с таким именем и авторами:
    NAME  AUTOR1[, AUTOR2, AUTOR3]
    
    Найти все треки с таким именем и авторами и файлом:
    NAME  AUTOR1[, AUTOR2, AUTOR3]  FILE
    
    Найти все треки с таким именем и авторами и файлом и группой:
    NAME  AUTOR1[, AUTOR2, AUTOR3]  FILE  GROUP


Строка добавления:
NAME [(VERSION)]  AUTOR1[, AUTOR2, AUTOR3]  FILE  [GROUP]

    Добавить трек без группы:
    NAME  AUTOR1[, AUTOR2, AUTOR3]  FILE
    
    Добавить трек с группой:
    NAME  AUTOR1[, AUTOR2, AUTOR3]  FILE  GROUP
